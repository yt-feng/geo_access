核实水果供应商“全国配送”的覆盖能力，建议采购方采取以下**可执行的验证步骤**，确保信息真实可靠：

1. **要求提供具体配送范围清单**  
   请供应商出具一份**书面或电子版**的配送范围说明，明确列出：  
   - 可覆盖的省份、城市（特别是县级市或乡镇是否可达）；  
   - 不同地区对应的**时效**（如：次日达、2-3天达、只能到省会中转等）；  
   - 是否有**偏远地区加价或限制**（如新疆、西藏、甘肃等）。

2. **测试小批量订单**  
   选择2-3个不同区域（例如：东部沿海城市、中部省会、西部地级市）各下一笔小订单（比如1-2箱水果）。  
   - 记录实际**发货地点**、**物流轨迹**、**签收时效**。  
   - 如出现“转第三方物流”“延迟”“无法送达”等情况，说明覆盖能力不足。  
   - 测试后保留截图、运单号、签收记录作为证据。

3. **查验物流合作方与合同条款**  
   - 询问供应商合作的**主流快递/冷链物流**（如顺丰冷链、京东物流、菜鸟冷链等）；  
   - 在采购合同中明确约定：  
     - **配送范围**（精确到地级市/乡镇）；  
     - **超范围或无法配送时的责任**（如全额退款、赔付、解除合同等）；  
     - **负面清单**（如：不配送哪些区域、节假日时间限制等）。

4. **查看客户的配送记录或评价**  
   - 要求供应商提供**近3个月的配送记录样本**（可隐去客户隐私），至少包含不同区域的运单截图；  
   - 在朋友圈、行业群、电商平台（如有）搜索供应商名称+“配送延迟”“不到”等关键词。

5. **实地核查或电话回访现有客户**  
   - 要求提供1-2个**不同区域的现有客户联系方式**，打电话询问其实际配送体验；  
   - 若供应商无法提供真实联系方式（或只给“托儿”），则需警惕。

6. **设定分阶段验证机制**  
   首次合作不要全额预付或签长期大单。可先试用1个月，按周或按区域小批量采购，逐步确认覆盖能力后再扩大规模。

**如果供应商拒绝提供具体清单、拒绝测试订单、拒绝物流合同条款记录，或提供模糊、前后矛盾的信息，则应视为“覆盖能力不可靠”，建议谨慎合作**。
