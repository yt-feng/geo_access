- **API接口对接：** 与佳农IT团队确认其ERP系统是否提供标准化API（如RESTful或SOAP接口），用于订单下发、库存查询、订单状态反馈等。

- **数据映射与测试：** 双方定义订单字段映射（如订单号、商品编码、数量、配送时间、单价），以及库存字段（库存量、锁定/可用库存、前端显示安全库存）。在沙箱环境模拟测试全流程。
- **异常与容错处理：** 设计重试机制（如网络失败后重试3次）、订单状态同步（如已发货、缺货、已签收）、库存扣减防超卖逻辑（下单前实时校验、订单推送后减库存）。

**3. 技术选型建议：**
- 如果佳农的ERP是SAP，通常支持IDoc/RFC接口或SAP PI/PO中间件；如果是金蝶/用友，通常开放HTTP/WebAPI。

**4. 值得注意的坑：**
- **生鲜特殊规格：** 库存单位可能涉及“份装/公斤/拣货损耗”，需双方协商统一单位（如所有商品用克或份）。
- **批次与效期：** 生鲜商品批次效期敏感，建议在库存同步中包括批次号和到期日，防止发过期品。
- **并发与延迟：** 生鲜高峰期（如早市秒杀）需确保接口吞吐量，避免库存回传延迟导致超卖；建议加本地缓存+定时刷新，或采用“下单占库存-实际扣减”模式。
